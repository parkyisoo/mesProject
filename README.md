# MES-Lite (경량 제조실행시스템)
> **"제조 현장 운영 흐름을 직접 설계하고 구현한 경량 MES 포트폴리오"**

---

### 기술 스택
* **Language:** Java 17
* **Framework:** Spring Boot 3.2.3
* **Database:** MySQL 8.0, JPA (Hibernate)
* **View:** Thymeleaf, Chart.js (KPI 시각화)

---

### 개발 기간
* **2026.03 (약 2주)**
* 제조 현장의 흐름을 직접 설계하고 DB 스키마부터 UI까지 1인 개발 진행

---

### 핵심 구현 기능
* ** 역할 기반 접근 제어 (RBAC)**
  * ADMIN / OPERATOR / QC 권한별 메뉴 노출 분기 처리
* ** 생산 프로세스 관리**
  * 작업지시 생성 및 상태(Open/Progress/Done) 관리
  * **State Machine** 방식을 도입한 LOT 추적 및 이력 자동 기록
* ** 실적 및 품질 관리**
  * 생산 실적 입력 (수량 검증 로직 및 LOT 자동 완료 처리)
  * 불량 원인 분류 및 처리 방법(폐기/재작업) 기록 기능
* ** 대시보드 및 KPI 리포트**
  * Chart.js를 활용한 수율, 불량률, UPH 시각화
  * 실시간 현황 요약 및 불량률 5% 초과 시 경고 알림

### 📝 프로젝트 배경 및 동기
* **현장의 소리:** 베트남에서 2년간 코딩 강사로 근무하며, 현지 제조업체에 취업한 지인들로부터 생산 현장의 생생한 이야기를 들었습니다. 거대 공장임에도 불구하고 많은 데이터가 **수기 장부나 파편화된 엑셀**로 관리되어, 불량 발생 시 원인 추적(Traceability)이 어렵고 실시간 수율 파악이 늦어지는 문제를 목격했습니다.
* **직무 목표:** **하나마이크론 비나(Hana Micron Vina)**의 MES 운영 직무 공고를 보며, 반도체 패키징처럼 정밀한 공정일수록 **데이터 가시성(Visibility)**이 곧 생산성과 직결됨을 깨달았습니다. 이를 깊이 있게 이해하기 위해 시스템의 흐름을 직접 설계하고 구현하는 프로젝트를 시작했습니다.
* **개발 철학:** 초기에는 로컬 환경에서 DB 스키마와 비즈니스 로직을 탄탄히 설계하는 데 집중했으며, 어느 정도 구현이 완료된 시점에 GitHub를 통해 프로젝트 관리를 시작했습니다.

### 🛠 기술 스택
* **Language:** Java 17

* **Database:** MySQL 8.0, JPA (Hibernate)
* **View:** Thymeleaf, Bootstrap 5, Chart.js (KPI 시각화)

---

### ✨ 핵심 구현 기능
* **🔐 역할 기반 접근 제어 (RBAC)**
  * ADMIN / OPERATOR / QC 권한별 메뉴 노출 및 접근 제한 처리
* **📊 생산 프로세스 관리**
  * 작업지시(Work Order) 생성 및 상태(Open/Progress/Done) 관리
  * **State Machine** 방식을 도입하여 LOT 상태 전이(Waiting → Progress → Completed) 추적
* **📈 실적 및 품질 관리**
  * 생산 실적 입력 시 수량 검증 로직 및 LOT 자동 완료 처리
  * 불량 원인(설비, 작업자 등) 분류 및 처리 방법(폐기, 재작업) 기록
* **🖥 대시보드 및 KPI 리포트**
  * Chart.js를 활용한 종합 수율(Yield), 불량률, UPH 시각화
  * 실시간 현황 요약 및 불량률 5% 초과 시 경고 알림
